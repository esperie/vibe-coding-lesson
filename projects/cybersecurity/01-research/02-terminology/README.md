## Quick Reference: Essential Terms for Day One

### Must-Know Acronyms

| Acronym | Full Form | Description |
|---------|-----------|-------------|
| **SOC** | Security Operations Center | Team monitoring security events 24/7 |
| **SIEM** | Security Information and Event Management | Central platform for log collection and analysis |
| **IOC** | Indicator of Compromise | Evidence that a breach has occurred |
| **TTP** | Tactics, Techniques, and Procedures | How adversaries operate |
| **CVE** | Common Vulnerabilities and Exposures | Unique identifier for known vulnerabilities |
| **IR** | Incident Response | Process of handling security breaches |
| **APT** | Advanced Persistent Threat | Sophisticated, long-term attack campaigns |
| **EDR** | Endpoint Detection and Response | Security tool monitoring endpoints |

### Team Colors

| Team | Role | Focus |
|------|------|-------|
| **Red Team** | Offensive security | Simulates attacks to find weaknesses |
| **Blue Team** | Defensive security | Protects systems and responds to threats |
| **Purple Team** | Collaborative | Bridges red and blue for continuous improvement |
| **White Team** | Oversight | Manages and referees security exercises |

### Incident Response Phases

```
1. Preparation    --> Pre-incident planning and readiness
2. Detection      --> Identifying potential security incidents
3. Containment    --> Limiting damage and spread
4. Eradication    --> Removing the threat completely
5. Recovery       --> Restoring systems to normal
6. Lessons Learned --> Post-incident review and improvement
```
